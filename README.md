# Villa Laura Website

Villa Laura is a low-maintenance guest-support website for a holiday house in Sardinia, Italy. It is not a booking platform. Bookings stay on Airbnb, while the site gives guests quick access to house guidance, YouTube walkthroughs, contact help, and a small curated gallery.

## Stack

- Static HTML generated by a small Node build script
- CSS and a tiny amount of vanilla JavaScript
- `sharp` for image optimisation during build
- Ready for Cloudflare Pages with `dist/` as the output directory

## Run Locally

```bash
npm install
npm run build
npm run preview
```

Then open the local URL printed by `serve`.

## Project Structure

- `site.config.json`: editable content, links, videos, and gallery source list
- `assets/source/`: curated original images committed to the repo
- `src/styles.css`: all styling
- `src/app.js`: small scroll reveal behaviour
- `scripts/build.mjs`: generates the final static site in `dist/`
- `public/`: static files copied directly to `dist/`

## Update Text, Videos, or Links

Edit `site.config.json`.

Useful fields:

- `links.airbnb`
- `links.whatsapp`
- `links.youtubeChannel`
- `hero`
- `about`
- `guideSections`
- `videos`
- `gallery`

After editing, run:

```bash
npm run build
```

## Update Photos

1. Add new curated originals to `assets/source/`
2. Update the `gallery` array in `site.config.json`
3. Run `npm run build`

The build script automatically generates optimised `.webp` and `.jpg` files into `dist/assets/`.

## Deploy

### Cloudflare Pages

- Framework preset: `None`
- Build command: `npm run build`
- Build output directory: `dist`

If automatic GitHub deployment is connected in Cloudflare Pages, every push to the production branch can publish a new version.

## Security

- Do not commit credentials, notes, or local key files
- `Doc_Keys_Villa_Laura.txt` and similar private files are ignored by `.gitignore`
- Review `git diff --cached` before every future commit
